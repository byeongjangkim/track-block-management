## 6.2 차단현황도 (BlockMapPage) ★ 메인 페이지

│                 │  줌 컨트롤 (우상단 절대 배치):      │

│                 │                                    │
│ 차단명령 목록   │                                    │

### 사이드바 구성 요소

| 요소 | 조건 | 기능 |
|---|---|---|
| 날짜 선택 | 항상 | 선택 날짜 기준 차단명령 조회 |
| 노선 필터 | 항상 | 차단명령 목록 노선 필터링 |
| 조직 선택 드롭다운 | `system_superuser`만 | 선택 시 해당 조직 viewport로 이동 |
| [지도 설정 ▼] | 항상 | 클릭으로 토글 |
| └ 관할 구간 표시 | 지도 설정 내 | 해당 조직 관할 구간 D3 오버레이 |
| └ 시설물 표시 | 지도 설정 내 | 표시 중인 노선 전체 시설물 레이어 ON/OFF |
| └ 노선 레이어 토글 | 지도 설정 내 | 고속/보통/지하철 그룹별 show/hide |
| 차단명령 목록 | flex-1, 스크롤 | 날짜 기준 전체 차단명령 |

### 차단명령 목록 배지

| 배지 | 조건 | 의미 |
|---|---|---|
| `지도` (파란색) | user geometry 있는 노선 + 구간 | 지도 위 오버레이로 표시 중 |
| `미표시` (주황색) | user geometry 없는 노선 | km 보간 불가 → 지도 미표시 |

### 두 가지 쿼리 패턴

| 쿼리 | 엔드포인트 | 용도 |
|---|---|---|
| 사이드바 목록 | `GET /api/v1/block-orders?date_from=&date_to=` | DB 전체 차단명령 |
| 지도 오버레이 | `GET /api/v1/map/block-orders/segments` | user geometry 있는 것만 GeoJSON |

### D3.js 지도 (RailwayMap)

- 투영: `d3.geoMercator().fitSize([W, H], allGeo)`
- 줌: `d3.zoom()`, scaleExtent `[0.5, 30]`
- 줌 임계값: `ZOOM_STATION=0.8` / `ZOOM_SEGMENT=3` / `ZOOM_DETAIL=8`
- 선 굵기: `stroke-width: 1.5`, `vector-effect: non-scaling-stroke`
- 노선 레이어: `source='user'` 실선 / `source='shp'` 점선 (`stroke-dasharray: 4 3`)
- 관할 경계: `stroke-width: 4`, `opacity: 0.7`, 분야별 색상
- 차단구간: UP `#ef4444` / DOWN `#f97316`, stroke-width 5

## 6.3 차단명령 (BlockOrdersPage)

- 경로: `/block-orders`
- 접근: 전체 조회, `org_admin` 이상 등록·수정·삭제

### 레이아웃

```
소속조직* [전체▼]  노선 [전체▼]  분야 [전체▼]  기간 [YYYY-MM-DD] ~ [YYYY-MM-DD]
                                                  (오늘)             (오늘+7일)
                              [초기화]  [조회]  [PDF 일괄등록*]  [+ 차단명령 등록*]

소속 조직 배지 + 총 건수
```

> \* `org_admin` 이상만 표시

### 필터 영역 UX 규칙

- **1줄 구성:** 레이블을 컨트롤 왼쪽에 인라인 배치
- **높이 통일:** 모든 `<select>`, `<input type="date">` 높이를 `h-9`으로 고정
- **select 너비:** `w-32`로 통일, `appearance-none` + 커스텀 `▾` 화살표
- **배경 없음:** 필터 영역 별도 카드 박스 없음
- **버튼 정렬:** [초기화] [조회] [+ 차단명령 등록] 오른쪽 끝

```
테이블:
  소속조직* | 노선 | 방향 | 구간(km)/단전구간 | 작업일자 | 시간 | 분야 | 차단종류 |
  작업책임자 | 안전관리자 | 외부 | 문서 | [수정|삭제]*
  (* superuser만 소속조직 열 / 본인 조직 org_admin만 수정·삭제)

  구간 표시: start_km/end_km 있으면 "368.5 ~ 451.8", 없으면 section_note ("청도SP~밀양SS") 또는 "—"
  행 클릭 → /block-map?date=YYYY-MM-DD
  문서·수정·삭제 셀 → stopPropagation
```

### 필터 항목

| 필터 | 타입 | API 파라미터 | 기본값 |
|---|---|---|---|
| 소속 조직 | 드롭다운 (superuser만) | `organization_id` | 전체 |
| 노선 | 드롭다운 | `route_id` | 전체 |
| 분야 | 드롭다운 (시설/전기/건축) | `field` | 전체 |
| 시작일 | 날짜 입력 | `date_from` | 오늘 |
| 종료일 | 날짜 입력 | `date_to` | 오늘+7일 |

### 조회 동작 규칙

- **[조회] 버튼 클릭 시에만** API 호출 (입력값 변경 즉시 호출 금지)
- **[초기화]:** 모든 필터를 기본값으로 복귀
- 두 단계 상태: `input*` (편집 중) / `applied*` (실제 조회 반영)

### 등록/수정 폼 (BlockOrderForm)

- 표시 방식: 오버레이 모달 (`fixed inset-0 bg-black/50`)
- **헤더:** 제목 + [시행문 PDF 불러오기] 버튼(신규 등록 시만) + [✕] 닫기
- **소속 조직 필드:** superuser → 드롭다운, org_admin → 자기 조직명 읽기 전용
- **시행문 PDF 불러오기:** 파일 선택 → pdfplumber 파싱 → 필드 자동 채움 → 신뢰도 < 0.6 경고 배너
  - 파싱 대상: 작업일자, 노선, 작업구간(km), 분야, 차단종류, 작업책임자, 철도운행안전관리자, 동원장비 여부

### PDF 일괄등록 (PdfImportModal)

- 진입: BlockOrdersPage 상단 [PDF 일괄등록] 버튼 (org_admin 이상)
- 대상: KORAIL 세부내역 PDF (◐각 열차 사이 차단◐ / ◐선로일시사용중지◐ / ◐전차선 단전◐ 섹션 포함)

**3단계 흐름:**

| 단계 | 내용 |
|---|---|
| 1. PDF 업로드 | 파일 선택 → `POST /bulk-parse` → 파싱 중 스피너 |
| 2. 내용 검토 | 파싱된 행 테이블 표시. 주황(`needs_review=true`)/파란(`section_note`) 강조. 행 삭제 가능 |
| 3. 저장 완료 | `POST /block-orders/bulk` → "N건 저장, M건 실패" 결과 배너 |

**검토 테이블 열:** 날짜 | 노선 | 방향 | 구간/단전구간 | 시간 | 분야 | 차단종류 | 작업책임자 | [삭제]

- `section_note` 셀: 파란 텍스트 (`text-blue-600`)로 변전소 구간명 표시 (예: `청도SP~밀양SS`)
- `needs_review` 행: 주황 배경 강조 (`bg-orange-50`)

> - 조직 행 클릭 or [관리 →] → 해당 조직 구간 편집 화면 (현재 구현과 동일)

```
[조직 선택 ▼]  총 N건                       [+ 구간 추가]

알림 배너 (성공/오류, 4초 자동 소멸)

┌────────┬────────────┬────────┬────────┬──────────────┐
│ 노선   │ 분야       │시작 km │종료 km │ 작업         │
├────────┼────────────┼────────┼────────┼──────────────┤
│ 경부선 │ 전체(행정) │ 368.5  │ 451.8  │ [수정] [삭제] │
│ 동해선 │ 전체(행정) │   0.0  │  88.97 │ [수정] [삭제] │
│ [▼]   │ [▼]        │ [입력] │ [입력] │ [저장] [취소] │ ← 편집 행
└────────┴────────────┴────────┴────────┴──────────────┘
```

### 필드 구성

| 필드 | 입력 방식 | 값 |
|---|---|---|
| 노선 | `<select>` | 전체 노선 목록 |
| 분야 | `<select>` | 전체(행정=all) / 시설 / 전기 / 건축 |
| 시작 km | `<input type="number" step="0.1">` | KORAIL 거리정 |
| 종료 km | `<input type="number" step="0.1">` | KORAIL 거리정 |

### 동작 규칙

- **조직 선택 시:** 해당 조직의 전체 관할구간 목록 표시
- **+ 구간 추가:** 테이블 최상단에 입력 행 삽입 (인라인 편집)
- **수정:** 해당 행을 편집 행으로 전환
- **삭제:** `window.confirm()` 확인 후 삭제
- **중복 방지:** 동일 (조직+노선+분야) 조합 등록 시 백엔드 400 오류 배너 표시
- **유효성:** 노선 미선택 또는 시작km ≥ 종료km 시 저장 차단

### 분야 배지 색상

| 분야 | 배경 | 텍스트 |
|---|---|---|
| 전체(행정, all) | `bg-gray-100` | `text-gray-600` |
| 시설 | `bg-indigo-100` | `text-indigo-700` |
| 전기 | `bg-yellow-100` | `text-yellow-700` |
| 건축 | `bg-emerald-100` | `text-emerald-700` |
